# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev       # Start dev server at http://localhost:5173
npm run build     # Production build to /dist
npm run preview   # Preview production build locally
```

No linting or testing framework is configured.

## Architecture

**React 18 + Vite PWA** — a local-first fitness tracker with optional Supabase sync and Claude AI coaching.

### Storage layer (`src/utils/storageHelper.js`)
All reads/writes go through a `window.storage` abstraction (Claude.ai API) that falls back to `localStorage`. This polyfill is installed in `main.jsx` before the app mounts. Every save triggers a **non-blocking** background sync to Supabase when configured. Storage keys: `workout-logs`, `daily-feelings`, `nutrition-logs`, `progress-photos`, `app-settings`, `workout-metadata`.

### App initialization (`src/App.jsx`)
On mount: load settings → init Supabase → fetch remote data → merge with local → render. All data-loading is async; components receive data via props from `App.jsx`.

### AI Coach (`src/components/AICoach.jsx`)
Calls the Claude API directly from the frontend. Sends a large system prompt containing Rubén's full profile (age, weight, goals, Mounjaro medication, nutrition targets, weekly split) plus the last N workout logs and daily feelings. The profile/context is hardcoded in this file.

### Supabase schema (`src/utils/database.js`)
Tables: `workouts` (exercises as JSONB), `feelings`, `nutrition`, `photos` — all with `user_id + date` indexes and row-level security. Supabase credentials are stored in `app-settings` (via the Settings UI), not in `.env`.

### Weekly training plan
Hardcoded in `App.jsx`: Push (Mon) / Pull (Tue) / Legs (Wed) / Pump (Thu) / Football/HIIT (Fri) / Rest (Sat) / Paddle optional (Sun).

### Deployment
Hosted on Netlify. `netlify.toml` sets Node 18, build command `npm run build`, output `dist/`, and SPA redirect (`/*` → `/index.html`). PWA manifest and service worker are generated by `vite-plugin-pwa`.
